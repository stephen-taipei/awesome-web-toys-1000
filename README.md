## 專案介紹

用 AI 輔助開發方式，打造純前端有趣的小工具專案。所有 toys 皆為純前端架構，無需後端與資料庫，每個專案皆為可獨立運作、具有視覺互動性、能帶來驚喜感的小作品。

### 核心特點

- **純前端架構**：無需後端伺服器、無資料庫依賴
- **完全獨立**：每個 toy 都是自包含的獨立網頁
- **視覺互動**：強調視覺效果與使用者互動體驗
- **開源免費**：所有程式碼皆以繁體中文註解並開放使用

## 技術架構

### 主要渲染技術

| 技術 | 用途 | Toys 數量 |
|-----|------|----------|
| Canvas 2D | 2D 繪圖、粒子系統、圖片處理 | ~350 |
| WebGL | 3D 渲染、著色器、高效能 2D | ~250 |
| WebGPU | 新一代高效能運算與渲染 | ~100 |
| SVG | 向量動畫、圖表繪製 | ~150 |
| CSS/DOM | 動畫特效、UI 效果 | ~100 |
| 混合技術 | 多技術整合運用 | ~50 |

### 音訊技術

- **Web Audio API**：音訊合成處理
- **AudioWorklet**：高效能音訊處理
- **MediaRecorder**：錄音功能

### 輸入 API

- **Pointer Events**：滑鼠、觸控統一處理
- **Gamepad API**：遊戲控制器支援
- **DeviceOrientation**：裝置方向感測
- **MediaDevices**：攝影機、麥克風存取

### 第三方函式庫

```javascript
// 3D 渲染
- Three.js         // WebGL 3D 場景
- Babylon.js       // 遊戲級 3D 引擎
- Regl             // 輕量 WebGL

// 2D 渲染
- PixiJS           // 高效能 2D
- p5.js            // 創意程式設計
- Konva            // Canvas 框架

// 物理引擎
- Matter.js        // 2D 物理
- Cannon.js        // 3D 物理
- Rapier           // Rust/WASM 物理

// 動畫
- GSAP             // 專業動畫
- Anime.js         // 輕量動畫
- Lottie           // 設計師動畫

// 音訊
- Tone.js          // 音樂合成
- Howler.js        // 音效播放

// 數據視覺
- D3.js            // 數據驅動圖表
- Chart.js         // 圖表繪製
- Vis.js           // 網絡圖
```

## 貢獻指南

1. Fork 本專案
2. 選擇一個未完成的 toy 進行開發
3. 在 `toys/` 目錄下建立對應資料夾
4. 確保 toy 符合規格
5. 更新 `plan.md` 標記完成進度
6. 提交 Pull Request

### 開發規格

- 每個 toy 必須可獨立運作
- 包含 `index.html`、`style.css`、`script.js`
- 提供 `preview.gif` 或 `preview.png` 預覽圖
- 使用繁體中文註解並開放使用
- 確保在主流瀏覽器上正常運行

### Prompt 模板

#### 首次開發 (從 #1 開始)

```markdown
## 專案背景
- 專案名稱：Awesome Web Toys 1000
- 專案目標：開發 1000 個純前端互動 Web Toys
- 技術棧：HTML + CSS + Vanilla JS / Canvas / WebGL / Three.js
- 參考文件：README.md, plan.md

## 當前進度
- 已完成：#0 (專案初始化)
- 當前任務：#1

## 本次任務
請開發第 #1 個 Web Toy：[Toy 名稱]

### 需求規格
- Toy 分類：[生成藝術/物理模擬/音樂視覺 等]
- 核心體驗：[描述互動方式與視覺效果]
- 輸入方式：[滑鼠/鍵盤/觸控/感測器]
- 輸出類型：[視覺/音效/可下載成品]
- 渲染技術：[Canvas 2D/WebGL/SVG/CSS]

### 技術要求
- 純前端實現，獨立 HTML 檔案可運作
- 流暢的 60fps 動畫效能
- 支援行動裝置觸控操作
- 包含基本音效 (如適用)
- 視覺效果具有驚喜感

### 完成標準
1. 核心互動體驗實作完成
2. 視覺效果調整完成
3. 行動裝置適配
4. 建立預覽圖 (GIF/PNG)
5. 提交 commit 並建立 PR

### PR 格式要求
- 標題：`[#1] Toy 名稱 - 分類`
- 內容包含：玩法說明、技術實現、預覽 GIF、操作方式

### 完成後請提供
1. 本次變更摘要 (3-5 點重點)
2. 關鍵演算法/特效說明
3. 效能優化技巧
4. 下一個 Toy #2 的前置準備建議
5. **上下文摘要** (供下次對話使用，約 200-300 字)
```
